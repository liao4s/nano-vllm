KEY FINDINGS
═══════════════════════════════════════════════════════════════════════════════

1. THE PROBLEM
   ───────────
   GatedDeltaNet uses Python dict to store per-sequence state:
   - self._recurrent_states[seq_id] = state_tensor
   - self._conv_states[seq_id] = conv_tensor
   
   CUDA graphs cannot re-execute Python code during replay():
   ✗ Dict lookups fail
   ✗ Tensors created at different GPU addresses
   ✗ Graph references stale memory locations
   
   Current workaround: engine forces enforce_eager=True
   → Disables CUDA graphs → ~3× slower decode (100 tok/s vs 300 tok/s)

2. CURRENT WORKAROUND (model_runner.py, lines 45-53)
   ──────────────────────────────────────────────────
   ```python
   model_type = getattr(hf_config, 'model_type', '')
   if model_type == 'qwen3_5_moe' and not self.enforce_eager:
       print("[model_runner] WARNING: Qwen3.5 uses stateful linear attention layers "
             "incompatible with CUDA Graph. Forcing enforce_eager=True.")
       self.enforce_eager = True
       config.enforce_eager = True
   ```

3. ROOT CAUSE: CUDA Graph Constraint
   ───────────────────────────────────
   CUDA graphs require ALL GPU memory addresses to be FIXED at capture time.
   
   FIXED = Pre-allocated, never reallocated, same address throughout.
   
   Dict-based state violates this: Each tensor created with NEW address each time.

4. REFERENCE: How Attention Handles This Successfully
   ──────────────────────────────────────────────────
   ✓ Pre-allocates k_cache and v_cache ONCE at fixed GPU addresses
   ✓ Stores to fixed cache using slot_mapping (which slots to write to)
   ✓ Uses flash_attn_with_kvcache with dynamic context_lens and block_tables
   ✓ Updates context values BEFORE graph.replay()
   ✓ Kernels read from SAME GPU addresses with UPDATED context values
   
   Code: nanovllm/layers/attention.py (lines 33-75)

TENSOR OPERATIONS (Per-Token Recurrent Rule):
──────────────────────────────────────────────
g_t = g[..., i].exp()                                    # Decay factor [1,32,1,1]
q_t = query[..., i]                                      # [1,32,128]
k_t = key[..., i]                                        # [1,32,128]
v_t = value[..., i]                                      # [1,32,128]

Operation 1: Decay
  recurrent_state *= g_t                                 # [1,32,128,128] *= [1,32,1,1]

Operation 2: Fetch from memory
  kv_mem = (recurrent_state * k_t.unsqueeze(-1)).sum(-2) # [1,32,128]

Operation 3: Delta
  delta = (v_t - kv_mem) * beta_t.unsqueeze(-1)          # [1,32,128]

Operation 4: Update state
  recurrent_state += k_t.unsqueeze(-1) * delta.unsqueeze(-2) # [1,32,128,128]

Operation 5: Output
  output = (recurrent_state * q_t.unsqueeze(-1)).sum(-2)  # [1,32,128]

SOLUTION SPECIFICATION
═══════════════════════════════════════════════════════════════════════════════

REPLACE DICT-BASED STATE WITH PRE-ALLOCATED BUFFERS:
────────────────────────────────────────────────────

BEFORE (Dict-based, incompatible):
  self._recurrent_states: dict[int, Tensor] = {}
  self._conv_states: dict[int, Tensor] = {}

AFTER (Buffer-based, compatible):
  self.recurrent_state_buffer: Tensor[512, 32, 128, 128]  # Fixed GPU address
  self.conv_state_buffer: Tensor[512, 8192, 3]            # Fixed GPU address

MEMORY REQUIREMENTS (Qwen3.5):
──────────────────────────────
Per linear attention layer:
  - Recurrent buffer: 512 × 32 × 128 × 128 × 4 bytes (float32) = 1.0 GB
  - Conv buffer:      512 × 8192 × 3 × 4 bytes = 50 MB
  - Total per layer:  ~1.05 GB

For 10 linear layers (Qwen3.5 has 10 linear attention layers):
  - Total: ~10.5 GB (or 5.2 GB with bfloat16, 50% reduction)

OPTIMIZATION:
Use bfloat16 instead of float32 (matches model's dtype, 50% memory savings)

STATE ACCESS PATTERN (Current):
──────────────────────────────
During prefill: Save to dict[seq_id]
During decode: Fetch from dict[seq_id], update dict[seq_id]

STATE ACCESS PATTERN (Proposed):
────────────────────────────────
Mapping: Create seq_id → buffer_idx before graph.replay()

During prefill:
  1. buffer[seq_id].zero_() (or buffer[buffer_idx])
  2. Compute
  3. buffer[buffer_idx].copy_(final_state)
  4. ALSO save to dict[seq_id] for later use

During decode (BEFORE graph.replay()):
  For each seq_id in batch:
    Copy dict[seq_id] → buffer[buffer_idx]

During graph.replay():
  Kernels read/write from buffer[buffer_idx] at FIXED GPU address

During decode (AFTER graph.replay()):
  For each seq_id in batch:
    Copy buffer[buffer_idx] → dict[seq_id]

KEY INSIGHT:
  Graph operates on FIXED buffer addresses
  But we still keep dict for state persistence across decode steps
  Dict ↔ Buffer copy happens CPU-side (outside graph)

CONCLUSION
═══════════════════════════════════════════════════════════════════════════════

The core issue is CUDA Graph's requirement for FIXED GPU memory addresses
throughout the inference session. GatedDeltaNet's dict-based state violates
this by creating new tensors with different addresses each time.

Solution: Replace dict with pre-allocated buffers (like k_cache/v_cache).

This enables 3× decode speedup via CUDA graphs, at the cost of ~5-10 GB
of pre-allocated GPU memory per model instance.

The implementation is straightforward (5 phases) but requires careful
coordination between ModelRunner (buffer management) and GatedDeltaNet
(state access).

Reference implementation already exists in Attention class — the pattern
is well-established and proven to work.
